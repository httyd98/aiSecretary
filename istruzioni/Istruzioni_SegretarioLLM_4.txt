## Inviare Messaggi — l'analogia

Finora abbiamo costruito solo la **ricezione** — il citofono che suona. Ora costruiamo la **voce** — come il sistema risponde verso WhatsApp.

Inviare un messaggio è più semplice della ricezione: è una normale **HTTP POST** che tu fai verso i server Meta, con il testo e il destinatario. Non aspetti che qualcuno ti chiami — sei tu che chiami.

---

## La funzione base di invio

```python
import httpx
import os

WHATSAPP_TOKEN = os.getenv("WHATSAPP_TOKEN")        # Token Meta
CLIENT_PHONE_NUMBER_ID = os.getenv("CLIENT_PHONE_NUMBER_ID")   # ID Numero A
PROF_PHONE_NUMBER_ID = os.getenv("PROF_PHONE_NUMBER_ID")       # ID Numero B
PROF_WA_ID = os.getenv("PROF_WA_ID")  # Numero WhatsApp del professionista

async def send_whatsapp_message(
    to: str,
    text: str,
    phone_number_id: str = None  # Se None, usa Numero A (clienti)
) -> bool:

    # Di default manda dal Numero A (bot clienti)
    # Se il destinatario è il professionista, usa il Numero B
    sender_id = phone_number_id or CLIENT_PHONE_NUMBER_ID

    url = f"https://graph.facebook.com/v18.0/{sender_id}/messages"

    payload = {
        "messaging_product": "whatsapp",
        "recipient_type": "individual",
        "to": to,
        "type": "text",
        "text": {"body": text}
    }

    headers = {
        "Authorization": f"Bearer {WHATSAPP_TOKEN}",
        "Content-Type": "application/json"
    }

    async with httpx.AsyncClient() as client:
        response = await client.post(url, json=payload, headers=headers)

    if response.status_code == 200:
        return True
    else:
        print(f"Errore invio: {response.status_code} - {response.text}")
        return False
```

## Gestire i tipi di messaggio

WhatsApp supporta più tipi di contenuto. Per il tuo MVP ne servono principalmente tre:

### Testo semplice — già visto sopra

### Testo con formattazione
WhatsApp usa una sintassi propria, diversa dal Markdown:

```python
# Grassetto   → *testo*
# Corsivo     → _testo_
# Barrato     → ~testo~
# Monospace   → ```testo```

async def send_formatted_summary(to: str, messages: list):
    lines = ["*📋 Riassunto messaggi di oggi*\n"]

    for msg in messages:
        lines.append(f"👤 *{msg['client_name']}*")
        lines.append(f"_\"{msg['last_message']}\"_")
        lines.append(f"🕐 {msg['timestamp']}\n")

    await send_whatsapp_message(to, "\n".join(lines))
```

### Messaggi con pulsanti interattivi
Utile per far rispondere il professionista con un tap:

```python
async def send_message_with_buttons(
    to: str,
    body: str,
    buttons: list[dict]  # [{"id": "yes", "title": "✅ Sì"}, ...]
):
    payload = {
        "messaging_product": "whatsapp",
        "to": to,
        "type": "interactive",
        "interactive": {
            "type": "button",
            "body": {"text": body},
            "action": {
                "buttons": [
                    {"type": "reply", "reply": {"id": b["id"], "title": b["title"]}}
                    for b in buttons[:3]  # Max 3 pulsanti
                ]
            }
        }
    }
    # ... stesso invio HTTP di prima
```

**Esempio pratico** — quando arriva un messaggio urgente, il professionista riceve:

```
⚠️ Mario Rossi chiede un appuntamento urgente per domani.

[✅ Gestisco io]  [🤖 Rispondi tu]  [⏰ Rimanda]
```

## Il problema della finestra 24 ore

Questa è una limitazione critica di WhatsApp Business API che devi conoscere:

```
┌─────────────────────────────────────────────────────┐
│  REGOLA META                                        │
│                                                     │
│  Puoi rispondere liberamente SOLO entro 24 ore      │
│  dall'ultimo messaggio del cliente.                 │
│                                                     │
│  Dopo 24 ore → puoi inviare SOLO template           │
│  pre-approvati da Meta.                             │
└─────────────────────────────────────────────────────┘
```

È come la regola di un ufficio postale: se il cliente ti ha scritto ieri, puoi rispondergli come vuoi. Se è passata una settimana, devi usare moduli ufficiali pre-approvati.

Per gestirlo:

```python
from datetime import datetime, timedelta

async def send_safe_message(to: str, text: str, conversation_id: int):

    # Controlla l'ultimo messaggio del cliente
    last_client_msg = await db.fetchrow("""
        SELECT timestamp FROM messages
        WHERE conversation_id = $1 AND role = 'client'
        ORDER BY timestamp DESC LIMIT 1
    """, conversation_id)

    if not last_client_msg:
        print("Nessun messaggio precedente, non posso inviare")
        return

    window_open = (
        datetime.now() - last_client_msg["timestamp"] < timedelta(hours=24)
    )

    if window_open:
        await send_whatsapp_message(to, text)
    else:
        # Usa un template pre-approvato
        await send_template_message(to, template_name="ricontatto_base")
```

## Mettere tutto insieme — il server completo

Ora uniamo webhook + database + AI + invio in un unico file funzionante:

```python
from fastapi import FastAPI, Request, HTTPException
from contextlib import asynccontextmanager
import asyncpg, os

# ── Setup ──────────────────────────────────────────
app = FastAPI()
db = None

@asynccontextmanager
async def lifespan(app: FastAPI):
    global db
    db = await asyncpg.create_pool(os.getenv("DATABASE_URL"))
    yield
    await db.close()

app = FastAPI(lifespan=lifespan)

# ── Webhook GET — verifica iniziale Meta ───────────
@app.get("/webhook")
async def verify(hub_mode: str, hub_verify_token: str, hub_challenge: str):
    if hub_mode == "subscribe" and hub_verify_token == os.getenv("VERIFY_TOKEN"):
        return int(hub_challenge)
    raise HTTPException(status_code=403)

# ── Webhook POST — messaggi in arrivo ──────────────
@app.post("/webhook")
async def receive(request: Request):
    body = await request.json()

    try:
        change = body["entry"][0]["changes"][0]["value"]

        if "messages" not in change:
            return {"status": "ignored"}  # È una notifica di stato

        message = change["messages"][0]
        phone_number_id = change["metadata"]["phone_number_id"]
        sender = message["from"]
        text = message.get("text", {}).get("body", "")
        message_id = message["id"]

        # Evita duplicati
        exists = await db.fetchval(
            "SELECT id FROM messages WHERE wa_message_id = $1", message_id
        )
        if exists:
            return {"status": "duplicate"}

        # Smista al handler corretto
        await process_incoming_message(phone_number_id, sender, text, message_id)

    except Exception as e:
        print(f"Errore elaborazione: {e}")

    # Rispondi sempre 200 a Meta, anche in caso di errore
    # Altrimenti Meta riprova all'infinito
    return {"status": "ok"}
```

│  [POST /webhook] ← messaggi in arrivo                │
│         │                                            │

│         │        └── summary     → genera + invia    │

│                  ├── legge DB (direttive + storia)   │
│                  ├── genera risposta                 │
│                  ├── send_message → cliente          │
│                  └── notifica professionista?        │
│                                                      │

        ↑                              ↓
